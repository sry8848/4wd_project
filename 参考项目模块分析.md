## 1. 电机控制 / 基础 GPIO

### 模块名称

电机控制 / 基础 GPIO 控制

### 对应源码怎么找

在以下文件中搜索：

- `SDC.py`
- `FIN.ipynb`
- `FaceColor.py`
- `All.py`
- `Rescue.py`
- `鎬讳唬.py`

关键词：

```text
IN1
IN2
IN3
IN4
ENA
ENB
def init
def run
def back
def left
def right
def spin_left
def spin_right
def brake
```

### 主要功能

控制小车前进、后退、左转、右转、原地左旋、原地右旋、停止。  
多数文件使用 TB6612FNG 类似的双路电机驱动逻辑，通过 GPIO 方向引脚和 PWM 使能引脚控制左右电机。

### 主要入口函数 / 类

- `init()`
- `run(leftspeed, rightspeed)`
- `back(leftspeed, rightspeed)`
- `left(leftspeed, rightspeed)`
- `right(leftspeed, rightspeed)`
- `spin_left(leftspeed, rightspeed)`
- `spin_right(leftspeed, rightspeed)`
- `brake()`
- `my_end()`：主要出现在 `FIN.ipynb`，用于结束时停止 PWM / GPIO。

### Python 依赖库

- `RPi.GPIO`
- `time`

### 系统或硬件依赖

- 树莓派 GPIO
- 电机驱动板
- 左右电机
- PWM 输出能力

### 需要的外部文件

无。

### 硬编码配置

常见 BCM 引脚：

```text
IN1 = 20
IN2 = 21
IN3 = 19
IN4 = 26
ENA = 16
ENB = 13
```

常见 PWM 频率：

```text
2000 Hz
```

### 与当前 Yahboom 4WD 小车迁移时的风险

- 当前小车的实际引脚可能与参考代码不一致。
- 电机方向可能相反，需要单独校准。
- 左右轮速度可能不一致，需要调 PWM。
- 部分源码最后写成 `GPIO.cleanup`，缺少括号，实际不会执行清理；应改为 `GPIO.cleanup()`。
- `key = 8` 与 `BuzzerPin = 8` / `buzzer = 8` 在部分代码中复用同一引脚，可能冲突。

### 是否适合优先迁移

高。

### 简短原因

这是所有移动功能的底层基础，但必须先做低速、短时间、可停止的单独测试，不能直接运行整合主程序。

- 避障函数会间接触发小车运动，不能直接运行整合版本。

## 4. A* 路径规划

### 模块名称

A* 路径规划

### 对应源码怎么找

在以下文件中搜索：

- `SDC.py`
- `FIN.ipynb`
- `All.py`
- `Rescue.py`

关键词：

```text
class Node
class node
class search_node
heuristic
astar
A_star
grid
maze
path_out
format_path
get_input
```

### 主要功能

在二维网格地图上规划从起点到终点的路径。  
`SDC.py` / `FIN.ipynb` 使用 `heapq` 实现较清晰的 A*；`All.py` / `Rescue.py` 是更早期的版本，代码更复杂。

### 主要入口函数 / 类

- `Node`
- `heuristic(a, b)`
- `astar(grid, start, end)`
- `grid_to_string(grid)`
- `get_input(prompt)`
- `format_path(path)`
- `A_star()`
- `search_node`
- `node`

### Python 依赖库

- `heapq`
- `math`：旧版 A* 中使用。

### 系统或硬件依赖

无。

### 需要的外部文件

无。

### 硬编码配置

`SDC.py` / `FIN.ipynb` 中存在硬编码网格，例如：

```text
5x5 grid
'X' 表示障碍物
'A' 表示可通行
```

旧版 `All.py` / `Rescue.py` 中有：

```text
6x6 maze
0 表示可通行
1 表示障碍物
```

### 与当前 Yahboom 4WD 小车迁移时的风险

- 算法本身可迁移性较好，但坐标系要和实际循迹网格一致。
- `A1` 这类输入坐标需要明确对应实际地图的哪一格。
- 算法输出的是网格路径，不是连续运动轨迹，需要路径执行层转换成前进/转向动作。
- 动态障碍更新时，障碍坐标的判断必须准确，否则会重规划到错误位置。

### 是否适合优先迁移

高。

### 简短原因

纯算法模块，不依赖硬件，最适合先迁移并用本地 Python 测试。


### 与当前 Yahboom 4WD 小车迁移时的风险

- 全景图片传输使用端口 `16666`，但参考项目中未看到完整对应的电脑端接收拼接服务。
- 文件路径依赖 `/home/pi/`，但当前推荐项目目录是 `/home/pi/4wd_project`。
- 摄像头相关代码可能阻塞主流程。

### 是否适合优先迁移

中。

### 简短原因

适合独立测试，但与主流程、电机无关；应先确认摄像头编号和 TCP 协议。

# 迁移时的全局注意点

1. 不要直接运行 `SDC.py`、`FIN.ipynb`、`All.py`、`Rescue.py`、`鎬讳唬.py` 这类整合入口。
2. 先迁移纯算法和只读传感器测试，再接入电机动作。
3. 所有 GPIO 程序必须补齐 `try / finally` 和 `GPIO.cleanup()`。
4. 参考源码中的真实邮箱、授权码、API Key、AppSecret、openId 等敏感配置不得复制。
5. 所有路径应从 `/home/pi/...` 改为当前项目目录下的相对路径或配置项。
6. 所有 GPIO 引脚必须以 Yahboom 当前小车实际接线为准，不能直接信任参考项目。
7. 摄像头编号需要现场确认，不能默认 `cv2.VideoCapture(1)` 可用。
8. `参考项目/` 应保持只读，迁移时把需要的逻辑重新整理到新模块中。
