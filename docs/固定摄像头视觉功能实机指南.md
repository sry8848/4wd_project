# 固定摄像头视觉功能实机指南

## 1. 当前硬件结论

当前 Sanhao Face USB 摄像头在摄像头云台转动时会直接掉线，因此本项目不再提供任何摄像头云台转动、扫描或自动寻找目标入口。

人脸、颜色、二维码和普通拍照都使用固定安装角度。识别类任务仍然从摄像头连续读取视频帧，不是只识别一张预先保存的图片。

保留的舵机和全景能力边界：

- `src/hardware/servo.py`：通用舵机硬件类，继续保留。
- `src/config.py` 中 `SERVO_PIN = 23`：超声波舵机配置，继续保留。
- `src/tools/test_panorama_180_car_turn.py`：通过小车原地转向拍摄全景，继续保留；它不转动摄像头云台。

## 2. 测试前检查

树莓派上的 `python` 指向 Python 2.7，以下命令必须使用 `python3`。

先停止后端和摄像头预览，避免多个进程同时占用 USB 摄像头：

```bash
sudo systemctl stop 4wd-camera.service
pgrep -af 'stream_camera_mjpeg|uvicorn|test_.*(photo|face|color|qr)'
sudo fuser -v /dev/video*
```

核对稳定设备路径当前指向：

```bash
readlink -f /dev/v4l/by-id/usb-lihappe8_Corp._Sanhao_Face-video-index0
```

若命令没有输出，说明 Linux 当前没有识别到摄像头。此时不能改用某个猜测的 `/dev/videoN` 掩盖问题。

## 3. 固定视角普通拍照

```bash
python3 src/tools/test_single_photo.py \
  --backend opencv \
  --device-path /dev/v4l/by-id/usb-lihappe8_Corp._Sanhao_Face-video-index0 \
  --width 640 \
  --height 480 \
  --output captures/fixed_camera_photo.jpg
```

通过标准：命令成功退出，照片真实存在，画面方向能同时覆盖起点乘客站位和前方障碍物区域。

## 4. 固定视角人脸登记与识别

登记仍使用命令行，第一版建议每人至少采集 10 张清晰正脸：

```bash
python3 src/tools/test_face_recognition.py enroll Alice \
  --device-path /dev/v4l/by-id/usb-lihappe8_Corp._Sanhao_Face-video-index0 \
  --count 10 \
  --timeout 20
```

固定视角连续识别：

```bash
python3 src/tools/test_face_recognition.py recognize \
  --device-path /dev/v4l/by-id/usb-lihappe8_Corp._Sanhao_Face-video-index0 \
  --threshold 0.30 \
  --confirm-frames 3 \
  --timeout 20
```

识别期间让主要人脸保持在固定画面内。工具不会转动摄像头寻找画面外的人。

## 5. 固定视角颜色识别

```bash
python3 src/tools/test_color_detect.py \
  --device-path /dev/v4l/by-id/usb-lihappe8_Corp._Sanhao_Face-video-index0 \
  --colors red,blue \
  --stable-frames 3 \
  --min-area 1500 \
  --timeout 15
```

工具从视频连续取帧，同一主色达到稳定帧数后停止并保存最终标注图。实际障碍物应使用大面积纯色块，避免反光和复杂纹理。

## 6. 固定视角二维码识别

```bash
python3 src/tools/test_qr_detect.py \
  --device-path /dev/v4l/by-id/usb-lihappe8_Corp._Sanhao_Face-video-index0 \
  --timeout 15 \
  --save-success-photo
```

收费站二维码必须使用半角冒号和大写 ASCII，例如：

```text
TOLL:GATE1
```

二维码需要正对摄像头、保留完整白色静区，并在约 20 cm 停车距离下占据足够画面宽度。程序不会转动摄像头寻找二维码。

## 7. 固定摄像头两分钟稳定性检查

依次确认普通拍照、人脸、颜色和二维码入口可用后，使用现有连续取帧工具保持摄像头打开至少 120 秒。检查：

- 摄像头设备节点没有消失。
- 没有黑屏、连续读帧失败或 USB 重新枚举。
- 全程没有执行任何摄像头舵机动作。
- 结束后能立即重新打开摄像头。

这项检查必须在树莓派实机完成，Windows 本地测试不能代替。

## 8. 车体转向全景

如需人工验证全景拍摄，只使用：

```bash
python3 src/tools/test_panorama_180_car_turn.py --frames 7 --stitch
```

该工具会驱动电机让小车原地转向。测试前必须架空车轮或清空周围区域，并准备立即断电。它不属于人脸、颜色和二维码自动流程，也不能在小车正常行程中并发运行。

## 9. 明确禁止的旧操作

- 不再使用摄像头 pan/tilt GPIO 配置。
- 不再传入 `--enable-servo-motion`、`--pan-angles` 或 `--tilt-angles`。
- 不再运行舵机二维码搜索、舵机全景或摄像头水平舵机测试脚本。
- 不要为了恢复旧命令而重新添加兼容参数；硬件结论已经证明该路径不可靠。
